# 故障排除指南

## 问题1：图片base64编码无效

### 症状
- 点击"开始识别"后显示错误
- 控制台提示 base64 编码无效

### 原因
1. 图片格式不支持
2. 图片读取失败
3. 数据格式错误

### 解决方案

**步骤1：重新编译小程序**
```
在微信开发者工具中按 Ctrl+B (Windows) 或 Cmd+B (Mac)
```

**步骤2：检查图片格式**
- 支持的格式：JPG、PNG、GIF、BMP
- 不支持：WebP、SVG 等

**步骤3：尝试其他图片**
- 使用不同的图片测试
- 确保图片文件完整无损

**步骤4：检查控制台日志**
- 在开发者工具底部查看 Console 输出
- 查找详细的错误信息

---

## 问题2：网络请求失败

### 症状
- 提示"网络请求失败"
- 无法调用OCR API

### 原因
1. 代理服务器未启动
2. 端口被占用
3. 网络连接问题

### 解决方案

**步骤1：启动代理服务器**
```
双击运行 start-server.bat
```

**步骤2：检查服务器状态**
- 应该看到："✓ OCR代理服务器已启动"
- 如果没有，检查是否有错误信息

**步骤3：检查端口占用**
- 如果提示端口被占用
- 关闭占用 3000 端口的程序
- 或修改 proxy-server.js 中的端口号

**步骤4：测试连接**
```
双击运行 test-connection.bat
```

---

## 问题3：代理服务器启动失败

### 症状
- 双击 start-server.bat 后显示错误
- 或者窗口立即关闭

### 原因
1. 未安装 Node.js
2. Node.js 版本过低
3. 文件路径问题

### 解决方案

**步骤1：安装 Node.js**
- 访问：https://nodejs.org/
- 下载 LTS 版本（推荐）
- 安装时勾选"Add to PATH"

**步骤2：验证安装**
打开命令提示符（CMD），输入：
```
node --version
```
应该显示版本号，如 v18.0.0

**步骤3：手动启动**
1. 打开 CMD
2. 进入 miniprogram 文件夹
3. 运行：`node proxy-server.js`

## 问题4：识别结果为空

### 症状
- API调用成功
- 但没有识别结果

### 原因
1. 图片中没有文字
2. 图片太模糊
3. 图片格式不支持

### 解决方案

**步骤1：检查图片**
- 确保图片包含清晰的文字
- 图片分辨率不要太低

**步骤2：尝试样例图片**
- 使用包含中文或英文的图片测试
- 确保文字足够大且清晰

**步骤3：查看响应数据**
- 在控制台查看完整的API响应
- 检查 error 字段和 msg 字段

## 问题5：微信开发者工具报错

### 症状
- 编译失败
- 页面无法加载

### 原因
1. 代码语法错误
2. 文件缺失
3. 配置错误

### 解决方案

**步骤1：检查错误信息**
- 查看底部的"问题"面板
- 定位具体的错误文件和行号

**步骤2：验证文件结构**
确认以下文件都存在：
```
miniprogram/
├── app.js
├── app.json
├── app.wxss
├── sitemap.json
└── pages/
    ├── index/
    │   ├── index.js
    │   ├── index.json
    │   ├── index.wxml
    │   └── index.wxss
    └── result/
        ├── result.js
        ├── result.json
        ├── result.wxml
        └── result.wxss
```

**步骤3：重新导入项目**
如果问题持续：
1. 关闭微信开发者工具
2. 重新打开
3. 重新导入 miniprogram 文件夹

## 调试技巧

### 1. 查看控制台输出
在微信开发者工具底部的 Console 面板查看：
- `选择的图片:` - 图片路径
- `图片转换成功，base64长度:` - base64编码长度
- `调用OCR API:` - API地址
- `API响应:` - 完整的API响应

### 2. 查看网络请求
在开发者工具中：
1. 点击顶部的"网络"选项卡
2. 查看请求详情
3. 检查请求和响应

### 3. 测试API连接
运行测试脚本：
```
双击 test-connection.bat
```

### 4. 验证代理服务器
检查代理服务器是否正常：
1. 打开浏览器
2. 访问 http://localhost:3000
3. 应该看到响应（即使错误也说明服务器在运行）

## 最后手段

如果一切都失败了，尝试：

1. **删除并重新创建项目**
   - 备份 miniprogram 文件夹
   - 删除项目
   - 重新导入

2. **使用最新版本的开发者工具**
   - 下载最新版本的微信开发者工具

3. **检查系统环境**
   - Windows：确保系统更新到最新
   - 确保有足够的磁盘空间

---

**提示**：大多数问题都可以通过重新启动代理服务器和重新编译小程序来解决。
